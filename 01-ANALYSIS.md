## 5. حالات الطلب

```
تسجيل أولي
    ↓
تحت المراجعة
    ↓
مقبول مبدئياً وبانتظار السداد
    ↓
سداد جزئي  ←→  مؤجل السداد
    ↓
مكتمل السداد
    ↓
طالب نشط

(في أي مرحلة):
    → مرفوض  ←(يسمح بإعادة التسجيل)
    → منسحب  ←(يسمح بإعادة التسجيل)
```

## 6. بنية API — Google Apps Script

### نقاط النهاية (Endpoints)

```javascript
GET  ?action=InitData          // الإعدادات + القوائم + الباقات + السنوات
GET  ?action=Students          // قائمة الطلاب (بدون ملفات)
GET  ?action=Student&id=...    // بيانات طالب واحد كاملة
GET  ?action=Records&id=...    // سجل إجراءات طالب
GET  ?action=Stats             // إحصائيات لوحة التحكم
GET  ?action=CheckDuplicate    // التحقق من تكرار التسجيل

POST ?action=Register          // تسجيل طالب جديد
POST ?action=UploadFile        // رفع ملف إلى Drive
POST ?action=UpdateStatus      // تحديث حالة الطالب
POST ?action=AddPayment        // تسجيل دفعة
POST ?action=CreateInvoice     // إنشاء فاتورة
POST ?action=AddNote           // إضافة ملاحظة
POST ?action=Login             // تسجيل دخول العاملين
POST ?action=SaveSettings      // حفظ الإعدادات
POST ?action=SavePackage       // حفظ/تحديث باقة
POST ?action=SaveYear          // حفظ/تحديث بيانات سنة
```

## 8. تسلسل العمل — التسجيل

```
1. المستخدم يفتح register.html
2. الواجهة تحمّل InitData من Apps Script
3. الواجهة تعرض الخطوة 1 من 5
4. المستخدم يملأ البيانات
5. في الخطوة 4 (رفع الملفات):
   - الواجهة ترسل الملف إلى Apps Script (POST UploadFile)
   - Apps Script يحفظه في Drive
   - يعود بـ: معرف الملف + رابطه
6. في الخطوة 5 (مراجعة وإرسال):
   - الواجهة تتحقق من التكرار (GET CheckDuplicate)
   - إذا مسموح: يرسل POST Register
   - Apps Script يُنشئ رقم الطلب تلقائياً
   - يحفظ البيانات في شيت الطلاب
   - يُسجل في سجل الإجراءات
   - يعيد رقم الطلب + رسالة نجاح
7. الواجهة تعرض رسالة النجاح مع رقم الطلب
```

## 9. تسلسل العمل — القبول والسداد

```
المشرف يفتح بطاقة الطالب
    ↓
يراجع الجواز والفيديو
    ↓
[قبول أولي] ←→ [رفض]
    ↓ (عند القبول)
Apps Script:
  - يحدث الحالة → "بانتظار السداد"
  - يُنشئ رقم فاتورة تلقائي
  - يحفظ في شيت الفواتير
  - يُسجل في سجل الإجراءات
    ↓
الواجهة تعرض:
  - رابط الفاتورة
  - زر "فتح واتساب" مع الرسالة الجاهزة
    ↓
(عند السداد)
المشرف يضغط: مكتمل / جزئي / مؤجل
    ↓
Apps Script يحدث: حالة الطالب + الفاتورة + سجل الإجراءات
```

## 10. آلية التحقق من التكرار

```
قبل حفظ التسجيل:
1. ابحث عن رقم الجواز في شيت الطلاب
2. ابحث عن جوال ولي الأمر
3. ابحث عن جوال الابن

إذا وُجد سجل:
  - تحقق من حالته
  - إذا كانت الحالة في: [تسجيل أولي, تحت المراجعة, بانتظار السداد, مكتمل السداد, طالب نشط]
    → ارفض التسجيل + أظهر رسالة مناسبة
  - إذا كانت الحالة في: [مرفوض, منسحب]
    → اسمح بإعادة التسجيل
```

## 12. تحسين الأداء — استراتيجية التحميل

```
عند فتح أي صفحة:
  Phase 1 (فوري):
    - تحميل InitData → الإعدادات + القوائم
    - عرض الهيكل الأساسي للصفحة

  Phase 2 (خلفي - بعد 100ms):
    - تحميل البيانات الثقيلة حسب الصفحة
    - تحديث الواجهة بشكل تلقائي عند الاكتمال

  في لوحة العاملين:
    Phase 1: InitData + عدد الإحصائيات فقط
    Phase 2 (بالخلفية): قائمة الطلاب المقيدة
    Phase 3 (عند الطلب): بيانات الطالب الفردية + مرفقاته

  Indexing:
    - بعد تحميل الطلاب: بناء Map<رقم_الطلب, بيانات>
    - بناء Map<جوال_الابن, رقم_الطلب>
    - بناء Map<جوال_ولي_الامر, رقم_الطلب>
    - تسريع البحث من O(n) إلى O(1)
```
